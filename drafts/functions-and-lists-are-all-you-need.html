Functions and lists are all you need

Published on 2020-06-20, under Programming, tagged with fp and ideas.

It seems that the building blocks of a programming language are lists and functions to work with them. Functions represent computations, an lists carry data around them.

What about if/then structure, we can use a function: true x y = x false x y = y ifthen bool t e = bool t e if bool t = ifthen bool t noop

What about loops, just use recursion loop fn x:xs = fn x ++ loop fn xs

With loops we can now work with lists: mostrar como map es un loop en realidad, como filter y reduce son en realidad un map

Null = [] Maybe x = [x]

lazyness (functions passed around), pasar una funcion por parametro, es curry pero con un extra al final de devolver una functon que no toma parametros pero llama a la que si

return () => original(1,2,3)

Exceptions? Dicts are lists of tuples and an index (of getters and setters) users = [['foo', 'bar'], ['baz', 'zaz']]

firstname user = user[0] lastname user = user[1]

So really firstname and lastname are aliases for get, we could just write get i obj = obj[i]

fistname = get 0 lastname = get 1

Objects hold data as state and methods to manipulate it. Methods are functions bound to objects: circle.draw() == draw(circle) Python went further in it's zen of explicit vs implicit and passes an explicit reference of the object (self)

By composing functions and piping data through we don't need inheritance.
