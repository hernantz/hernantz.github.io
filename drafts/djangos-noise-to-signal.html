Django's noise to signal

Published on Wed 12 August 2015, under Programming, tagged with django, python and testing.

Las seniales de django son una buena idea cuando queres responder a un evento de algun modelo. Si el evento es generado por un modelo que es tuyo, entonces es mejor no escribir una senial y usar directamente algun queryset manager u clase controladora que se encargue de vincular todas las piezas que se mueven

Si el modelo que genera la senial no es parte de tu codigo, sino de alguna libreria o app de terceros, entonces tal vez sea una buena idea.

Es lo digo porque las seniales hacen posible perder el control de lo que esta pasando y de repente tenes comportamientos inesperados.

Ejemplo cuando se crea un Payment, en la senial post save, crear un invoice en stripe Ahora los tests van a tener que todo el tiempo estar mockeand stripe Avoid signals for code you own. Try to avoid it for code you don't own.
